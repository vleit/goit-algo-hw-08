# Аналіз алгоритмів видачі решти

## Жадібний алгоритм

- Підхід: вибираємо найбільший номінал монети, поки не закінчиться сума.
- Складність: O(n), де n — кількість номіналів монет.
- Переваги: дуже швидкий, простий.
- Недоліки: не завжди оптимальний для нестандартних наборів монет.

## Динамічне програмування

- Підхід: для кожної суми обчислюємо мінімальну кількість монет.
- Складність: O(amount × k), де k — кількість номіналів.
- Переваги: завжди знаходить оптимальне рішення.
- Недоліки: повільніше для великих сум.

## Порівняння

- Для набору монет `[50, 25, 10, 5, 2, 1]` обидва алгоритми дають однаковий результат для суми 113: `{50: 2, 10: 1, 2: 1, 1: 1}`.
- Жадібний алгоритм набагато швидший при великих суммах.
- DP гарантує оптимум, особливо для нестандартних наборів монет.
